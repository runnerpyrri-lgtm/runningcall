## [0.13.4] - 2026-07-10
출시 전 안정화: 검색·날씨 오류를 숨기지 않고 정직하게 알리도록 고쳤다.

### Fixed
- 위치 검색에서 카카오 API 장애(401/403/429·네트워크 오류·타임아웃)를 "결과 없음"(200)으로 위장하던 문제 수정.
  주소·키워드 두 검색이 모두 실패하면 502(`upstream_error`)를 반환하고, 실패·빈 응답은 CDN에 캐시되지 않도록
  `Cache-Control: no-store`를 적용했다. 정상적인 "결과 없음"은 짧은 캐시(60초)만 쓴다. 화면에는 "결과 없음"과
  구분되는 "위치 검색이 일시적으로 불안정해요" 안내가 뜬다.
- 대기질(pm2.5·pm10)·강수확률 결측을 0으로 뭉개 "공기 최상"으로 점수화하던 문제 수정. 결측은 null로 보존하고,
  점수 계산에서 해당 요인을 제외한 뒤 남은 가중치로 재정규화한다(러닝 골든마스터 점수는 불변). 대기질을 못 불러오면
  화면에 "대기질 정보를 불러오지 못했어요 — 점수에 반영되지 않았어요" 배지와 "정보 없음" 등급을 표시한다.
  대기질 API 장애가 예보 전체를 실패시키지 않도록 분리했다.

### Changed
- 위치 검색의 "동네명까지 입력" 게이트를 제거했다(회장님 승인). 이제 강남역 같은 역명, 도로명 주소, 건물명도
  모든 활동에서 검색된다. 등산 모드의 산 우선 정렬은 유지된다.
- README·배포 가이드·.env.example에서 "카카오 키가 없으면 Nominatim으로 폴백"이라는 옛 설명을 실제 동작
  (키 없으면 검색 비활성 + 안내 표시)으로 바로잡고, 사용하지 않는 AI_GATEWAY_* 항목을 제거했다.

### Added
- 검색 API 오류 경로 테스트 6종(두 레그 실패→502/no-store, 부분 성공, 빈 결과 캐시, 키 미설정→503)과
  대기질 결측 점수화 테스트를 추가했다.
- CI가 main 브랜치 push에도 실행되도록 트리거를 추가했다(main 상태 신호 상시 확보).
- 하루 2회(KST 06:00·18:00) 작은 개선 1개를 draft PR로 상신하는 자가개선 워크플로
  (.github/workflows/daily-self-improve.yml)를 추가했다. 발행(merge)은 사람이 한다.

## [0.13.3] - 2026-07-10
출시 전 위치 검색·외부 데이터 정책·알림 안내를 보강했다.

### Fixed
- Kakao 키가 없거나 실패했을 때 Nominatim의 무관한 장소를 위치 검색 결과로 보여주던 폴백을 제거했다. 키가 없으면 현재 위치 사용을 안내하며, 검색어 길이와 위도·경도 범위를 검증한다.
- Origin·Referer가 없는 직접 API 요청을 허용하던 검사를 `Sec-Fetch-Site`까지 포함한 same-origin 브라우저 요청으로 제한했다.
- Nominatim 역지오코딩 요청에 앱 식별자, 인스턴스별 1초 간격, 일 단위 응답 캐시를 적용했다.
- 정식 출시되지 않은 Notification Triggers 의존 코드를 제거하고 알림이 앱 실행 중에만 동작한다는 안내로 통일했다.
- 화면 하단에 Open-Meteo·OpenStreetMap 출처와 현재 위치 좌표 전달 범위를 표시했다.
- PostCSS를 취약점 수정 버전으로 고정하고 pnpm 버전·빌드 허용 설정을 명확히 했다.

## [0.13.2] - 2026-07-09
동네 이름 위치 검색이 전혀 안 되던 문제 수정.

### Fixed
- 위치 검색이 "성수동"처럼 한글로 끝나는 동네명을 입력해도 항상 "동네명까지 입력해 주세요"로 막히던 버그 수정.
  원인은 동네명 추출 정규식의 트레일링 `\b`(ASCII 단어 경계)로, JS에서 한글은 단어문자가 아니라 한글로 끝나면
  경계가 성립하지 않아 매치가 항상 실패했다(걷기·산책·러닝·자전거 등 hike 외 전 활동에서 검색 전면 차단).
  `(?![가-힣])` 부정 룩어헤드로 교체. `neighborhoodMatch`를 `lib/search.ts`로 이동하고 회귀 테스트 추가.

## [0.13.1] - 2026-07-07
"골든 카드" 성능·사용성 보완.

### Changed
- 애니메이션 전면 최적화(기기 불문 60fps 목표): 색종이를 DOM 노드에서 캔버스 파티클로 전환(스폰 렉 제거),
  홀로포일·버튼·1위 카드의 빛 스윕을 background-position에서 GPU transform으로 교체, 릴 회전의 blur 필터 제거,
  캔버스 DPR 1.5 상한·파티클 총량 260개 상한·비/눈 입자 수 축소, 광선에 will-change 지정, backdrop-filter 제거.
- 6지표 그리드의 강수 칸을 이해하기 어려운 "접이식"에서 "비올확률 %"로 변경.
- 추천 시간대 잭팟 버튼의 라벨을 정중앙 정렬(슬롯 이모지를 절대배치로 분리).
- 슬롯 릴 회전 중 화면을 탭하면 즉시 전체 순위를 공개하고, 완료 후 다시 탭하면 창이 닫히도록 변경.
- 추천 시간대 뽑기가 끝나면 멈춘 릴 대신 골드 완료 패널로 하단을 교체.
- 오늘 추천 시간대가 지났을 때 밋밋한 안내문 대신 골드 "마감" 카드(내일 뽑기 버튼 포함) 노출.

## [0.13.0] - 2026-07-07
"골든 카드" 가챠 리디자인 — 점수를 카드 개봉으로 공개하는 게임형 메인 화면.

### Changed
- 메인 히어로를 신호 다이얼에서 가챠 카드로 교체: 앱 진입 시 카드가 2.25초 회전하며 개봉되고,
  카드를 탭하면 다시 개봉. 활동·날짜·위치 전환은 스핀 없이 퀵 리빌.
- 점수 티어 시스템 도입: 80+ 골드 LEGENDARY(황금 번개 3연타·도장 슬램·분수 불꽃·풀스크린 금빛),
  70대 EPIC 보라 유성우, 60대 SUPER RARE 청록 오로라, 50대 RARE 블루 슈팅스타,
  40대 UNCOMMON 그린 팝, 30대 COMMON, 20대 GLOOMY(어둠·안개), 20 이하 DISASTER(경보·균열).
- 카드 안 날씨 씬은 실측 데이터 기반: 강수량·확률에 따른 빗줄기 밀도, 적설·저온의 눈(3겹 원근),
  확률·운량에 따른 해·구름, 폭우·돌풍 시 카드 내 번개.
- 티어별 랜덤 멘트 풀(80+는 칭찬만), 등급 확률 배지·별점·수집 일련번호(올해 N번째 카드).
- 추천 시간대를 골드 잭팟 버튼 + 슬롯머신 릴로 교체: 릴이 돌아 금·은·동 순위가 시상되고
  (금메달 대축포), 순위 카드에 핵심 지표 칩 3개(강수 필수) 표시. 순위 탭 = 알림 설정,
  빈 곳 탭 = 닫기. 추천이 없으면 기존 빈 상태 카드 유지.
- 전체 팔레트를 아스팔트 네이비에서 가챠 다크퍼플 + 골드로 교체(신호등 등급색은 유지).

### Unchanged
- 점수 계산·날씨 파이프라인(lib) 미수정 — 러닝 골든마스터 테스트 19개 그대로 통과.
