## 결정 사항

### 1. 히어로 조건 칩 — 모바일 2열 그리드로 분리
- 모바일(≤620px 등)에서 `.status-chips`를 히어로 텍스트 옆에 끼워 넣지 않고, **점수+문구 아래 별도 줄에 2열 균등 그리드**로 배치. 데스크탑은 기존 유지.
- 구현: 모바일 브레이크포인트에서 `.status-chips { display:grid; grid-template-columns: repeat(2, 1fr); }`로 전환. 마크업 위치는 그대로(hero-copy 안)이되 CSS만 반응형 분기.

### 2. 애견산책 플랜 심화 — reason/alternative/careTip
`DogPlan`에 필드 추가:
```ts
export type DogPlan = {
  signal: "go" | "caution" | "avoid";
  signalText: string;
  reason: string;        // 신호 판정 이유 ("비 소식 + 점수 낮음" 등)
  alternative: string;   // 자제일 때 대안 ("배변만 짧게 / 실내 노즈워크")
  careTip: string;       // 다녀온 후 케어 ("발 말리기" 등, 상황별)
  walkLength: string;
  paw: PawRisk;
  checklist: string[];
};
```
`getDogPlan`에서 signal 판정 근거(비/더위/미세먼지/발바닥 중 가장 나쁜 요인)로 reason 생성. avoid/caution일 때 alternative 채움. careTip은 비/더위/추위 등 상황별 다르게.

### 3. 전체 운동 일지 (다이어리) — 신규 핵심 기능
- 저장: `running-alarm:journal:v1` = `Record<string, { activities: ActivityKey[]; note: string }>` (날짜 "YYYY-MM-DD" 키).
- **월 캘린더**: 날짜 셀마다 그날 기록된 활동들의 이모지(최대 3개, 넘으면 "+N") 표시.
- **날짜 클릭 → 상세 패널**(같은 화면 하단, 모달 아님): 5개 활동 체크(토글, 다중 가능) + **한 줄 메모 입력창**(저장 버튼 또는 blur 시 자동 저장).
- 활동 체크는 기존 `activity-log`(v0.4.1)와 **연동**: 일지에서 체크하면 해당 활동의 로그에도 그 날짜가 추가/제거됨(단일 진실 소스는 activity-log, journal은 그 날의 note만 별도 저장 + 캘린더 조회 뷰).
- 진입: PC 좌측 레일 하단에 **"📔 운동 일지"** 버튼(활동 목록과 분리된 별도 항목). 모바일은 드로어 메뉴에 "📔 운동 일지" 항목(기존 활동별 기록 항목과 별개로 유지, 혹은 대체 — 전체 일지가 상위 개념이므로 드로어엔 "운동 일지"만 남기고 기존 "{활동} 기록 보기"는 제거).
- 신규 컴포넌트: `JournalView` (page.tsx), 데이터 계층: `lib/journal.ts`.

### 4. 좌측 레일 — 큰 글자만
- `ActivityRail`의 `rail` variant에서 tagline(`<small>`) 제거. **이모지 + 활동명만**, 폰트 확대.
- 레일 하단에 구분선 + "📔 운동 일지" 버튼 추가(활동 선택과 별개 동작 — 클릭 시 일지 뷰로 전환).

### 5. `activity-tagline` 제거
- 탭/레일 아래의 "🏃 달리기 좋은 순간을 짚어드려요" 한 줄 삭제. 탭 선택 즉시 내부 탭(오늘판단/준비/기록/가이드) 노출.

### 6. 등산 데이파트 — 3구간 통일 (저녁 제거)
- `HIKE_DAY_PARTS`에서 저녁(16~19) 제거 → **새벽(4~7)·아침(8~10)·낮(11~15)** 3구간. 다른 활동과 동일한 1줄 그리드(`cols-3`)로 통일. 하산 안전 철학(어두워지기 전 하산)과도 부합.
- 관련 CSS `cols-4` 규칙은 유지하되(혹시 재사용) 실제 hike는 3구간 렌더.

### 7. 동적 가이드 — 날씨/조건 맞춤
- 가이드 탭 상단에 **오늘 조건 맞춤 블록**을 조건부로 추가(예: 비 오면 "🌧️ 오늘은 비 소식 — 미끄럼 주의", 강풍이면 "💨 오늘 바람 강함 — ..." 등). 기존 정적 `ACTIVITY_GUIDE[activity].guide` 블록 **위에** 노출.
- 구현: `lib/activity-guide.ts`에 `getDynamicGuideBlock(activity, slot): GuideBlock | null` 함수 추가 — 현재 슬롯의 강수/바람/자외선/미세먼지/폭염 여부를 보고 활동별 맞춤 한 블록 생성. 조건 없으면 null(정적 가이드만 표시).

### 8. 활동별 위치 기억
- 신규 저장: `running-alarm:activity-location:v1` = `Record<ActivityKey, LocationPoint | undefined>`.
- 활동 전환 시 그 활동에 저장된 마지막 위치가 있으면 그걸로 전환(없으면 현재 위치 유지). 위치를 바꾸면(검색/GPS) 현재 활동 키에 저장.
- **즐겨찾기(saved locations)는 전역 유지** — 활동별로 나누지 않음(사용자 자산은 공유, 마지막 활성 위치만 활동별 분리). 검색 모달에서 즐겨찾기 목록은 그대로 전체 노출.
