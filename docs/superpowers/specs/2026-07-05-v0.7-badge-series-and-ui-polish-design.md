# Part A — 뱃지 3000+ 시리즈 (v0.7.0)

## A1. 통계 확장 (lib/achievements.ts `AchievementStats`)

기존 필드 유지 + 추가:
```ts
durationByActivity: Record<ActivityKey, number>;      // 활동별 누적 시간(분)
maxSingleDurationByActivity: Record<ActivityKey, number>; // 한 번 최장 시간(분)
totalDistance: number;   // run+bike 누적 km
totalDuration: number;   // 전 활동 누적 분
```
`computeStats(journal, log)`가 일지 durationMin 합산으로 계산.

## A2. 시리즈 생성기

```ts
export type BadgeSeries = {
  id: string; group: string; emoji: string; name: string; unit: string;
  metric: (s: AchievementStats) => number;
  thresholds: number[]; // 오름차순, 초반 촘촘
  titleFor: (threshold: number) => string;
};
export const SERIES: BadgeSeries[];
```

**사다리 헬퍼** — 초반 밀도(1·2·3…) 후 점점 넓어짐:
```ts
function ladder(segments: Array<[from: number, to: number, step: number]>): number[];
// 예: bigLadder(max) = ladder([[1,50,1],[55,100,5],[110,300,10],[350,1000,50],[1100,max,100]])
```

**시리즈 목록(약 26개, group):**
- 누적 횟수: 총 운동(1000) · 걷기/애견/러닝/등산/자전거 횟수(각 500)
- 거리: 러닝(3000)·자전거(5000)·합계(8000) km
- 시간: 걷기/애견/러닝/자전거(각 6000)·등산(12000)·합계(30000) 분
- 꾸준함: 연속일(366)·한 달 최다일(31)
- 탐험: 서로 다른 산(100곳)
- 한 방: 최고 러닝 거리(42km 마라톤)·최고 자전거 거리(200)·활동별 최장 시간(각 600분)
- 마음: 일기 편수(500)·기분 기록일(500)·최고 기분일(366)
- 다양성: 하루 종목(2~5)·한 주 종목(2~5)·경험 종목(1~5)

**뱃지 생성:** 각 시리즈 × 각 threshold → `Badge { id: `${series.id}@${threshold}`, seriesId, group, emoji, title, tier, target: threshold, progress: series.metric }`. tier는 시리즈 내 위치 비율로 bronze→diamond 부여.

**개수 보장:** `BADGES.length >= 3000` (테스트). 미달 시 사다리 밀도/max 상향. 초반 밀도: 주요 시리즈의 thresholds가 `[1,2,3,4,5]`로 시작함을 테스트.

## A3. 시리즈 평가

```ts
export type SeriesProgress = {
  series: BadgeSeries; value: number; level: number; total: number;
  nextThreshold: number | null;  // 없으면 완주
  earnedIds: string[];
};
export function evaluateSeries(stats: AchievementStats): SeriesProgress[];
```
`evaluateBadges`/`newlyEarned`/seen는 그대로(전체 BADGES 기반). 성능: 3000회 비교는 무시할 수준.

## A4. 도전과제 화면 (컬렉션)

- 상단: 큰 `달성 N / 3xxx` + 진행 링(또는 슬림 바), 등급별 개수 요약.
- **시리즈 카드 그리드**: 이모지·시리즈명·`Lv {level}`·다음 목표 슬림 진행바·`다음: {next}{unit} · {level}/{total}`. 완주 시 "완주 ✓".
- 카드 탭 → **아코디언 펼침**: 그 시리즈 전체 뱃지 사다리(펼칠 때만 렌더). 달성=컬러, 다음 목표=강조, 이후=회색. NEW 표시.
- 그룹 헤더로 시리즈 묶음.

## A5. 축하 팝업 요약

한 번에 여러 뱃지 달성 시(초반 흔함): 상위 4개 크게 + `외 N개 달성!` 텍스트. 단수면 기존대로.

## A6. 테스트 (achievements.test.ts 갱신)

- `BADGES.length >= 3000`, id 중복 없음
- 주요 시리즈 초반 threshold가 [1,2,3,4,5]로 시작
- computeStats 확장 필드(durationByActivity·totalDistance·totalDuration·maxSingleDuration)
- evaluateSeries: level·nextThreshold·완주(null)
- 러닝 14km면 dist_run 시리즈에서 10km까지 레벨, 다음 15km
- newlyEarned/seen 동작 유지

# Part B — UI 고급 폴리시 (v0.7.1, 정체성 유지)

큰 리브랜딩 없이 완성도만 정교화. 브랜드색·레이아웃·구조 불변.

## B1. 디자인 토큰 (globals.css `:root`)
- 라운드 스케일 추가: `--r-sm:12px; --r-md:16px; --r-lg:20px; --r-xl:24px` 후 주요 카드에 일관 적용.
- 그림자 위계 정교화: 2단 소프트 섀도(`--shadow-sm`/`--shadow`/신규 `--shadow-lg`) 재정의(더 부드럽고 층지게).
- 표면 틴트: `--surface:#fafbfe; --surface-2:#f4f7fc` 도입, 섹션 배경에 미세 대비.

## B2. 마이크로 인터랙션
- 주요 인터랙티브 요소(활동 탭·레일·카드 버튼·시리즈 카드·기분/활동 칩)에 `:active { transform: scale(0.98) }` + `transition: transform .12s, box-shadow .15s`.
- 진행바(도전과제·시리즈)에 width transition, 링 애니메이션.

## B3. 타이포
- 헤딩 자간(`letter-spacing:-0.015em`) 정돈, 숫자 강조(히어로 점수·달성 수) tabular·굵기 위계 강화.

## B4. 카드·시트 폴리시
- 히어로/근거 카드/시트/도전과제 카드 그림자·테두리·여백 리듬 다듬기(정체성 유지 범위 내).
- 도전과제 화면을 새 폴리시의 쇼케이스로.

## B5. 검증
- 시각 회귀 없이 골든마스터·빌드 통과, 브라우저 360/1440에서 어긋남·오버플로우 0, 콘솔 0.
