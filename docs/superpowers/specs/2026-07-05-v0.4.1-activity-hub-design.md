## 설계

### 1. PC 좌측 카테고리 레일
- **데스크탑(≥1200px)**: 왼쪽 고정 세로 레일. 항목마다 이모지 + 활동명(전체 라벨) + 한 줄 tagline + 선택 상태 강조.
- **1080~1199px & 모바일**: 기존 상단 5탭. (레일 대신 탭.)
- 레이아웃: `.dashboard-frame`을 데스크탑에서 `[레일 210px][콘텐츠]` 2단 그리드로(최대폭 ~1320px), 콘텐츠 내부는 기존 2단(content-cols) 유지. 모든 폭에서 오버플로우 0 검증.

### 2. 활동 내부 탭 — 판단 / 준비 / 기록 / 가이드
활동 선택 아래에 세그먼트 컨트롤. 기본 = 판단. 활동 전환 시 판단으로 리셋.
- **판단**: 오늘/내일 토글 + 히어로 + 활동 안전카드(애견/등산/자전거) + 추천 시간대 + 아침낮저녁 + 일출일몰 + 오늘의 한마디 + 지표 6개. (= 현재 메인 화면.)
- **준비**: 복장 상세(OutfitPlan 인라인) + 활동 준비물 체크 + 주의사항. (기존 복장 시트 내용을 인라인으로.)
- **기록**: 현재 활동의 기록 달력·연속일·이번주/이번달·목표. (드로어 기록 → 이 탭으로 이전.)
- **가이드**: 현재 활동에 맞는 실전 팁(아래 콘텐츠).
- 모바일에서 복잡하지 않게 4개 세그먼트(가로 스크롤 없이 한 줄). 지표 탭 등 기존 시트(탭하면 상세)는 판단 탭에서 유지.

### 3. 활동별 기록 시스템
- 신규 저장: `running-alarm:activity-log:v1` = `{ walk: string[]; dog: string[]; run: string[]; hike: string[]; bike: string[] }` (날짜 "YYYY-MM-DD" 배열).
- 목표 저장: `running-alarm:activity-goals:v1` = `{ walk:5, dog:7, run:3, hike:2, bike:2 }` (기본값, 편집 가능).
- **마이그레이션**: 기존 `running-alarm:runlog`가 있고 신규 키가 없으면 → `run` 배열로 이관하고 신규 키 저장. 기존 `running-alarm:goal`은 run 목표로.
- 기록 탭은 **현재 선택 활동**의 기록만. 버튼 문구·이모지도 활동별("오늘 걸었어요 🚶", "오늘 산행했어요 ⛰️" 등).
- 활동별 기본 목표 문구: 걷기 주5·애견 주7·러닝 주3·등산 월2(=주1 환산)·자전거 주2.
- `lib/record.ts`(Set 기반 계산) 재사용. 신규 `lib/activity-record.ts`에 저장 shape·마이그레이션·기본값·목표라벨.

### 4. 활동별 실전 콘텐츠 (준비·가이드)
`lib/activity-guide.ts` 신설 — 활동별 준비물/주의/실전 팁. 얕은 문구 금지.
- **걷기**: 미세먼지 판단, 자외선, 오래 걷기 요령, 출퇴근 도보 팁.
- **애견산책**: 발바닥·지면열, 배변 산책, 노령견/소형견 주의, 계절별.
- **러닝**: 페이스, 습도, 워밍업, 회복, 조깅/인터벌 구분.
- **등산**: 하산 마감, 일출 산행, 낙뢰·돌풍, 조망, 아이젠·헤드랜턴.
- **자전거**: 맞바람·측풍, 젖은 노면, 라이트, 헬멧, 강변길.

### 5. 모바일 5탭 정리
- 짧은 라벨(걷기/산책/러닝/등산/자전거) 균등 한 줄, 활성 pill 또렷하게, 터치 타깃 넉넉히. 360~430px 검증.
