## 컨셉

등산은 **목적지(산) 중심** — "오늘 이 산 올라가도 돼?". 우리는 **등산 날씨 판단 앱**(지도·트래킹·코스DB 아님). 앱 전체는 러닝 전용에서 **야외활동(걷기·애견산책·러닝·등산·자전거) 전용**으로 확장.

## 🔒 불변식
- **러닝 골든마스터** — run 점수·톤·멘트 완전 불변. 새 날씨 필드는 slot 점수 미사용.
- localStorage 키 유지, 마지막 선택 활동 기억. 새 파일 첫 줄 한글 주석, 한글 마침표 종결.

## 확정된 순서
`ACTIVITY_ORDER = ["walk", "dog", "run", "hike", "bike"]` (걷기 → 애견산책 → 러닝 → 등산 → 자전거). 신규 기본 탭 = **walk**.

## 외부 AI 리뷰 반영 (반드시 보완)

### R1. 내일 일출·일몰 분리 저장 (중요 버그 방지)
현재 `RawForecast`는 오늘 sunrise/sunset만 저장. **내일 산행 하산 마감이 오늘 일몰로 계산되면 안 됨.**
→ `RawForecast`에 `sunriseTomorrow`, `sunsetTomorrow` 추가. `getHikePlan`은 `isTomorrow`면 내일 일출·일몰 사용.

### R2. 진짜 새벽 산행 (04시 시작)
- `getRankedWindows` 시작 시각: hike = **04시**, 그 외 = 06시. 종료 20시.
- hike 데이파트 4구간(겹침 없이): **새벽 04~07 / 아침 08~10 / 낮 11~15 / 저녁 16~19**.

### R3. 산 검색 판정 품질
- `대` suffix 제거(건국대·경희대 오판 방지). 산 판정 = 이름이 `산·봉·악·오름·고개·령`으로 끝나거나 카카오 `category_name`에 산·공원·관광·명소·자연 포함.
- `SearchResult`에 `kind`, `categoryName` 추가. `isMountain()`은 export해서 단위 테스트.

### R4. 앱 전체 확장 문구 (이름은 임시 유지, 설명은 확장)
- `app/layout.tsx`: title/description → "러닝콜 — 걷기·등산·산책·러닝·자전거, 나가기 좋은 시간".
- `app/manifest.ts`: name 유지, description 확장.
- 공유 문구·알림 제목·드로어 브랜드 부제 → 야외활동 확장형.
- (앱 이름 러닝콜은 리브랜딩 전까지 유지.)

### R5. 가이드/팁 정리
- `lib/guide.ts`: 러닝 전용 → **활동별 점수 기준 / 등산 전 체크 / 하산 시간 / 날씨 위험 신호** 중심으로 개편.
- `app/api/tip/route.ts`: 현재 미사용(AI 러닝 코치). 제거하고 관련 UI 링크 정리.

### R6. 5개 탭 모바일 최적화
- 탭 라벨은 **짧게**: 걷기 / 산책 / 러닝 / 등산 / 자전거 (dog는 탭에서 "산책", 전체 라벨 "애견산책"은 히어로·드로어·tagline에서). `ActivityProfile.short` 필드 추가.
- iPhone SE(360~375)·대형폰·iPad·갤탭·1440+ PC에서 5탭 넘침 0 검증. flex:1 균등 유지.

### R7. 테스트 강화
- hike 강수·강풍 민감도 / `getHikePlan` 낙뢰·돌풍·하산마감 / hike daypart 4구간 / `isMountain` 산 판정 / 러닝 골든마스터. 최소 5개 추가.

## 등산 종합 안전 카드 — `getHikePlan(...)`
하산 마감(일몰−2.5h, 내일이면 내일 일몰), 일출 산행(새벽), 안전 신호(⛈️낙뢰 code95+ / 💨돌풍 gust / 🌧️비노면 / ❄️결빙 / 🌬️능선바람), 👁️조망(visibility·cloud·미세), 정상 체감 안내, 준비물 체크리스트(상황별 헤드랜턴·아이젠 등).

## 🎯 카테고리별 "수요 밀착" 문구 (핵심 품질 목표)
각 활동 사용자가 **진짜 필요로 하는 것**을 캐치해 문구를 씀. 특히 **오늘의 한마디(composeOneLiner)는 조건별 3~4개로 확장**, 다양하고 와닿게.

- **🐕 애견산책** — 강아지를 아끼는 마음 + 실질 도움. 발바닥 화상, 더위·추위 취약성, 냄새 맡기·탐험의 즐거움, 물·배변봉투·발 닦기, 교감. 따뜻하고 진심 어린 톤 + 안전 실용.
  예: "오늘 같은 날은 강아지도 꼬리부터 흔들 거예요. 좋아하는 냄새 실컷 맡게 여유 있게 걸어주세요." / "지금 아스팔트는 발바닥에 화상이에요. 손등 5초 테스트 후 뜨거우면 해 지고 나가세요."
- **🚶 걷기** — 부담 없는 건강·기분 전환. 가볍게, 무리 없이, 바깥 공기, 기분.
- **🏃 러닝** — 페이스·컨디션·기록 + 부상 예방. (기존 유지·강화.)
- **⛰️ 등산** — 안전 제일. 하산 시간, 정상 방한·조망, 준비물, 능선 위험. 성취감 + 실용.
- **🚴 자전거** — 안전(바람·비 위험)·장비·거리. (기존 유지·강화.)

## 성공 기준
1. 러닝 골든마스터 통과. 새 필드가 run/걷기/애견/자전거 점수 불변.
2. 등산: 산 검색·오늘/내일 하산 마감(내일은 내일 일몰)·04시 새벽 시간대·안전 카드.
3. 오늘의 한마디가 카테고리별 3~4개로 다양·수요 밀착(특히 애견 진심+실용).
4. 앱 메타·공유·알림·가이드가 야외활동 확장형.
5. 5개 탭 전 기기 넘침 0, 등산 카드·4데이파트 반응형.
6. isMountain 등 테스트 통과, build 통과, 배포 성공.
