# v0.4.0 — 등산 카테고리 + 전면 업그레이드 설계

- 작성일: 2026-07-05
- 상태: 승인됨 (사용자 + 외부 AI 제안 통합)
- 범위: 등산(hike) 카테고리 신설(산 검색·등산 점수·하산/일출/낙뢰/돌풍/조망/준비물 안전 카드·새벽 시간대·날씨 데이터 확장) + 카테고리 순서/문구/이모지/설명 전면 정비 + 전 기기 반응형 UI.

## 컨셉

등산은 **목적지(산) 중심** — "지금 나가도 돼?"가 아니라 **"오늘 이 산 올라가도 돼?"**. 우리는 **등산 날씨 판단 앱**(지도·트래킹·코스DB 아님 — AllTrails 추종 금지). 기존 위치검색 엔진에 산 우선 검색을 얹어 목적지 산의 날씨를 등산 관점으로 판단.

## 🔒 불변식 (유지)

- **러닝 골든마스터** — run 점수·톤·멘트 완전 불변. 새 날씨 필드/hike 로직은 run 계산에 영향 없음.
- localStorage 키 유지, 마지막 선택 활동 기억.
- 새 파일 첫 줄 한글 주석, 한글 문장 마침표 종결.

## 결정 사항

### 1. 카테고리 순서 (수요순, 확정)
`ACTIVITY_ORDER = ["walk", "hike", "dog", "run", "bike"]` (걷기 → 등산 → 애견산책 → 러닝 → 자전거).
신규 사용자 기본 탭 = **walk**. (기존 사용자는 저장된 활동 유지.)

### 2. 등산 점수 프로필 (hike)
```
hike: {
  label: "등산", emoji: "⛰️", tagline: "산 오르기 좋은 시간과 하산 여유까지",
  weights: { dust:0.12, temperature:0.18, precipitation:0.28, uv:0.10, humidity:0.10, wind:0.22 }, // 합 1.00
  temp: { optimalLo:6, optimalHi:18, coldSlope:4.5, hotSlope:5 },
  heat: { hot1:31, hot1Cap:55, hot2:34, hot2Cap:38 },
  windCap: { speed:12, cap:38 },       // 능선 강풍 위험
  resultMode: "score",
  terms: { verbDo:"등산하", outfitTitle:"등산 준비물", alarmTitle:"산행 알림",
           rainyTag:"우중 산행", blockedTag:"산행 위험", nowTag:"지금 산행 딱!",
           notifReady:"산 오르기 좋은 시간이에요" }
}
```

### 3. 날씨 데이터 확장 (등산 안전용 — run 점수엔 미사용)
`lib/weather.ts` Open-Meteo 요청과 `HourlyInput`에 **선택 필드** 추가:
- `weatherCode`(WMO, 낙뢰 95~99 감지), `windGust`(wind_gusts_10m), `visibility`, `cloudCover`, `snowfall`.
- 기존 slot 계산은 이 필드를 **읽지 않음** → 골든마스터 불변. hike 안전 카드만 사용.
- HourlyInput 새 필드는 optional(`?`)로 하위호환.

### 4. 등산 종합 안전 카드 — `getHikePlan(...)` (lib/activity.ts)
애견 `getDogPlan`처럼 등산 전용 카드를 히어로 아래(좌측 열) 노출.
- **하산 마감**: 일몰 − 2.5h. "일몰 19:56 · 늦어도 17:30엔 하산 시작하세요".
- **일출 산행**(현재 hour ≤ 8 또는 새벽 슬롯 선택 시): 일출 시각 + "정상 일출 보려면 약 X시 출발".
- **안전 신호**(우선순위순, 최대 3~4개):
  - ⛈️ 낙뢰(weatherCode 95~99) → 위험, "능선·정상 낙뢰 매우 위험, 산행 미루기"
  - 💨 돌풍(windGust ≥ 14 또는 wind ≥ 12) → 위험/주의, "정상부 돌풍·저체온 위험"
  - 🌧️ 비/노면(precip ≥ 0.5 또는 prob ≥ 60) → 주의, "바위·흙길 미끄럼 주의"
  - ❄️ 결빙/눈(snowfall > 0 또는 기온 ≤ 0 & 습함) → 주의, "아이젠·스틱 챙기기"
  - 🌬️ 능선 바람(wind ≥ 9) → 주의, "정상부는 체감 더 낮아요"
- **조망**: visibility/cloudCover 기반 "오늘 조망 좋아요/흐려요"(미세먼지도 조망 관점 반영).
- **정상 체감 안내**: "정상부는 기온보다 6~10°C 낮아요. 겹쳐 입으세요"(정성적, 고도 정밀보정은 v0.5.0).
- **준비물 체크리스트**: 물·바람막이·보조배터리 기본 + 상황별(우비/헤드랜턴(일몰 근접·늦은 출발)/아이젠(결빙)/모자·선크림(자외선)/장갑(추위)).

### 5. 새벽 포함 시간대 (등산 전용)
- `getRankedWindows`: 시작 시각을 활동별로. hike = 5시부터(새벽 산행 포함), 그 외 = 6시.
- `getDayParts`: hike 전용 4구간 **새벽(5~8)·아침(8~11)·낮(11~15)·저녁(15~19)**. 그 외 활동은 기존 3구간(아침/낮/저녁) 유지. dayparts UI는 3/4개 모두 반응형(모바일 2×2, 넓은 화면 4열).

### 6. 산 우선 검색 (app/api/search-location + page.tsx)
- `SearchResult`에 `kind?: "address"|"place"|"mountain"`, `categoryName?: string` 추가.
- 카카오 키워드 검색 결과에서 category_name 보존. 산 판정: 이름이 산·봉·악·오름·고개·령으로 끝나거나 카테고리에 산·공원·관광명소 포함 → `kind:"mountain"`.
- **hike 활동일 때**: 검색창 placeholder "산 이름 검색 (예: 북한산)", 결과에서 산 우선 정렬, 표시 "북한산 · 서울 강북구 우이동". 그 외 활동: "동네·지역 검색".
- 검색 모달에 현재 activity 전달.

### 7. 카테고리 설명(tagline) 노출 + 문구/이모지 전면
- 5개 활동 tagline 갱신, **활동 탭 아래 한 줄(현재 활동 설명)** + **드로어 각 카테고리 밑 설명** 노출.
- 헤드라인/서브라인/한마디/브리핑/멘트/복장/지표설명에 hike 카피 추가 + 5개 활동 톤 통일. 등산 지표는 미세=조망, 바람=능선, 일몰=하산 관점으로.

### 8. 전 기기 반응형
- iPhone SE(375)~대형폰, 갤탭·아이패드(768~1024), 데스크탑·와이드(1280~1920+) 재검증.
- 신규(등산 카드·4시간대·산검색결과) 전부 반응형. 필요 breakpoint 추가. 데스크탑 2단 유지.

## 성공 기준
1. 러닝 골든마스터 통과(불변). 새 날씨 필드가 run/걷기/애견/자전거 점수 안 바꿈.
2. 등산 탭: 산 검색되고(북한산 등), 하산 마감·안전 신호·준비물 카드 표시, 새벽 시간대 추천.
3. 낙뢰/돌풍/조망 카드가 실제 날씨 데이터로 동작.
4. 5개 활동 어디서도 다른 활동 표현 안 샘(grep). 각 카테고리 설명 노출.
5. 모바일/태블릿/데스크탑 전부 오버플로우 0, 등산 카드·4시간대 정상.
6. test/build 통과, Vercel 배포 성공.
