# v0.6.0 — 성과 확인(캘린더 요약) + 도전과제/뱃지 시스템 설계

날짜: 2026-07-05
상태: 설계 승인(사용자 축하 연출·진입 위치 확정) — 스펙 정리

## 목표

운동 일지에 쌓인 기록을 "성취"로 되돌려주는 레이어를 추가한다. ① 캘린더에서 날짜를 누르면 그날 성과를 바로 요약해 보여주고 기록 있는 날은 빨간 링으로 표시한다. ② 일지 밑에 **도전과제(뱃지)** 카테고리를 신설해 약 36개 뱃지를 과거 기록 소급 적용으로 채우고, 새로 달성될 때마다 큰 축하 팝업으로 "모으는 맛"을 준다.

## 불변 조건 (Global Constraints)

- 러닝 골든마스터 통과 유지.
- 기존 데이터(journal v2 `running-alarm:journal:v2`, activity-log `running-alarm:activity-log:v1`)는 읽기만 — 스키마 변경 없음.
- 뱃지는 **과거 기록 소급**: 앱을 처음 열어도 이미 달성한 뱃지가 채워져 있음.
- 신규 파일 첫 줄 한국어 역할 주석.
- 검증: `pnpm test && pnpm typecheck && pnpm build` + 브라우저(360/1440).

## 1. 달성 엔진 (lib/achievements.ts, 신규)

순수 함수로 통계 계산 → 뱃지 판정. UI와 분리해 단위 테스트.

### 통계 계산
```ts
export type AchievementStats = {
  totalSessions: number;                        // 활동한 서로 다른 날 수(일지+기록탭 합집합)
  sessionsByActivity: Record<ActivityKey, number>;
  distanceByActivity: Record<ActivityKey, number>; // 일지 거리 합(run·bike)
  maxSingle: Record<ActivityKey, number>;          // 한 번 최고 거리(일지)
  uniqueMountains: number;                       // 서로 다른 산 이름 수(일지 hike place)
  bestStreak: number;                            // 최장 연속 활동일
  moodDays: number;                              // 기분 기록한 날
  greatDays: number;                             // 기분 "최고"인 날
  diaryEntries: number;                          // 일기(note) 작성한 날
  bestMonthDays: number;                         // 한 달 최다 활동일
  triedActivities: number;                       // 경험한 종목 수
  maxDayVariety: number;                         // 하루 최다 종목 수
  maxWeekVariety: number;                        // 한 주 최다 종목 수
};
export function computeStats(journal: Journal, log: ActivityLog): AchievementStats;
```
- 날짜별 활동 집합 = 일지 활동 키 ∪ 기록탭 날짜. `totalSessions`·streak·variety·sessionsByActivity는 이 합집합 기반.
- distance/maxSingle/mountains/mood/diary는 **일지에만** 존재 → 일지 기반.
- streak: 활동일을 정렬해 연속(하루 차이) 최장 길이.
- week 버킷: `Math.floor(epochDay / 7)`로 묶어 distinct 종목 수 최대.

### 뱃지 카탈로그
```ts
export type BadgeTier = "bronze" | "silver" | "gold" | "diamond" | "special";
export type Badge = {
  id: string; group: string; emoji: string; title: string; desc: string;
  tier: BadgeTier; target: number; progress: (s: AchievementStats) => number;
};
export const BADGES: Badge[];  // 약 36개
export type EvaluatedBadge = { badge: Badge; current: number; earned: boolean };
export function evaluateBadges(stats: AchievementStats): EvaluatedBadge[]; // current=min(progress,target)
export function earnedCount(evs: EvaluatedBadge[]): number;
export function newlyEarned(evs: EvaluatedBadge[], seen: string[]): Badge[];
```

카탈로그(그룹·id·목표·tier):
- **first**(target 1): first_any(전체), first_walk, first_dog, first_run, first_hike, first_bike, first_diary — special
- **streak**: streak_3(bronze)·streak_7(silver)·streak_14(gold)·streak_30(diamond)
- **total**(totalSessions): total_10(bronze)·total_50(silver)·total_100(gold)·total_200(diamond)
- **run**: run_dist_10·run_dist_50·run_dist_100(bronze/silver/gold, distanceByActivity.run), run_count_30(silver, sessionsByActivity.run)
- **single**: single_run_5·single_run_10(bronze/silver, maxSingle.run), single_bike_30(gold, maxSingle.bike)
- **bike**: bike_dist_100(silver)·bike_dist_300(gold, distanceByActivity.bike)
- **mountain**: mtn_3·mtn_5·mtn_10(bronze/silver/gold, uniqueMountains)
- **variety**: variety_day2(bronze, maxDayVariety), variety_week3(silver, maxWeekVariety), variety_all5(gold, triedActivities)
- **monthly**: month_10(silver)·month_20(gold, bestMonthDays)
- **heart**: great_5(bronze, greatDays), mood_10(silver, moodDays), diary_10·diary_30(silver/gold, diaryEntries)

총 7+4+4+4+3+2+3+3+2+4 = **36개**.

### 확인 상태 저장
```ts
export const BADGES_SEEN_KEY = "running-alarm:badges-seen:v1";
export function loadSeen(): string[];   // 손상 방어
export function saveSeen(ids: string[]): void;
```
- 최초 평가 시 seen 키가 없으면 **현재 달성분 전부를 조용히 seen 처리**(첫 실행에 팝업 폭탄 방지). 이후 새 달성만 축하.

### 테스트 (lib/__tests__/achievements.test.ts)
- computeStats: 합집합 세션 수, 러닝 누적 거리, 산 unique, streak(연속 3일), day/week variety, bestMonthDays
- evaluateBadges: 달성/미달성·current 클램프
- newlyEarned: seen 제외
- 빈 데이터 → 달성 0, first류 미달성

## 2. 캘린더 성과 요약 + 빨간 링 (JournalView)

- 상태 `editing`(로컬): 날짜 탭 시 그 날 기록이 **비어있으면 편집기**, **있으면 요약**(editing=false).
- **요약 패널**(캘린더 아래): `prettyDate`, 기분 이모지, 활동별 요약 줄("🏃 러닝 5.2km · 30분", "⛰️ 등산 · 북한산"), 일기 미리보기 + **✏️ 편집** 버튼. 다른 날 누르면 갱신.
- **✏️ 편집** → editing=true → 기존 편집기 화면. 저장/취소 → 캘린더 복귀.
- **빨간 링**: `.journal-cell.has-record`(기록 있는 날). 오늘은 파란 링 유지 → 오늘+기록이면 빨간 링 + 파란 점으로 구분.

## 4. 축하 팝업 (CelebrationModal) + 감지·저장
