# v0.5.0 — 운동 일지 심화(다이어리) + 준비/애견/레일 UI 폴리시 설계

날짜: 2026-07-05
상태: 승인됨(사용자 "업그레이드 시작해") — 구현 진행

## 목표

운동 일지를 "체크 + 한 줄"에서 **활동별 가벼운 기록 + 기분 + 자유 일기**를 쓰는 다이어리 앱 감성으로 끌어올린다. 동시에 캘린더 짤림·정렬 붕괴, 애견 체크리스트 두 줄, 준비 탭 밋밋함, 모바일 드로어 카테고리 장황함, 이모지 임팩트 부족을 함께 해결한다.

## 불변 조건 (Global Constraints)

- 러닝 골든마스터 통과 유지 (점수·톤·멘트 불변).
- 기존 운동 일지 데이터(`running-alarm:journal:v1`) **무손실 마이그레이션**.
- 등산 데이파트 4구간(새벽/아침/낮/저녁) 유지.
- 모든 신규 소스 파일 첫 줄에 한국어 역할 주석.
- 검증: `pnpm test && pnpm typecheck && pnpm build` + 브라우저(360/768/1440).

## 1. 운동 일지 데이터 구조 (lib/journal.ts, v1 → v2)

### 타입
```ts
export type MoodKey = "great" | "good" | "soso" | "tired" | "hard";
export const MOODS: Array<{ key: MoodKey; emoji: string; label: string }> = [
  { key: "great", emoji: "😄", label: "최고" },
  { key: "good",  emoji: "🙂", label: "좋음" },
  { key: "soso",  emoji: "😐", label: "보통" },
  { key: "tired", emoji: "😪", label: "지침" },
  { key: "hard",  emoji: "😣", label: "힘듦" }
];

export type ActivityRecord = {
  key: ActivityKey;
  distanceKm?: number;   // run, bike
  durationMin?: number;  // 전 활동
  place?: string;        // hike (다녀온 산)
};

export type JournalEntry = { activities: ActivityRecord[]; mood?: MoodKey; note: string };
export type Journal = Record<string, JournalEntry>;

export const JOURNAL_KEY = "running-alarm:journal:v2";
const LEGACY_KEY = "running-alarm:journal:v1";
```

### 활동별 입력 필드 (전부 선택 입력)
```ts
export type JournalField = {
  key: "distanceKm" | "durationMin" | "place";
  label: string; kind: "number" | "text"; unit?: string; placeholder?: string;
};
export const ACTIVITY_JOURNAL_FIELDS: Record<ActivityKey, JournalField[]> = {
  walk: [{ key: "durationMin", label: "시간", kind: "number", unit: "분" }],
  dog:  [{ key: "durationMin", label: "시간", kind: "number", unit: "분" }],
  run:  [{ key: "distanceKm", label: "거리", kind: "number", unit: "km" },
         { key: "durationMin", label: "시간", kind: "number", unit: "분" }],
  hike: [{ key: "place", label: "다녀온 산", kind: "text", placeholder: "예: 북한산" },
         { key: "durationMin", label: "시간", kind: "number", unit: "분" }],
  bike: [{ key: "distanceKm", label: "거리", kind: "number", unit: "km" },
         { key: "durationMin", label: "시간", kind: "number", unit: "분" }]
};
```

### 함수
- `emptyJournal()`, `emptyEntry(): JournalEntry`
- `normalizeJournal(value)`: v2(활동=객체) **및** v1(활동=문자열 배열) 두 형태 모두 수용. note는 string, mood는 유효값만, 숫자 필드는 유한·0 이상만, place는 trim·최대 40자.
- `loadJournal()`: v2 키 읽기 → 없으면 LEGACY_KEY(v1)에서 1회 마이그레이션(활동 `string[]`→`{key}[]`, note 유지, mood 없음) 후 v2로 저장 → normalize.
- `saveJournal(journal)`, `setEntry(journal, date, entry)`
- `isEntryEmpty(entry)`: 활동 없음 && note 공백 && mood 없음 → 저장 시 해당 날짜 삭제(스토리지 정리).

### 테스트 (lib/__tests__/journal.test.ts 갱신)
- v2 CRUD, isEntryEmpty
- **v1→v2 마이그레이션**: `{"2026-01-01":{activities:["run","hike"],note:"메모"}}` → 활동이 `{key:"run"}` 객체로, note 유지
- normalize: 손상 활동/음수 거리/과길이 place 정리, 잘못된 날짜 키 스킵

## 2. 운동 일지 화면 (JournalView, app/page.tsx)

두 내부 화면: **캘린더 ↔ 편집기**. `selectedDate === null`이면 캘린더, 아니면 편집기. draft는 JournalView 로컬 상태.

### 캘린더 화면
- 헤더: `📔 나의 운동 일지` + `이번 달 N일 기록`(해당 월 비어있지 않은 날 수)
- 월 이동(cal-nav) 유지
- 셀: **고정 크기**, 활동 이모지 **크게(≈1rem)** 최대 3개 + `+N`, 셀 안에서 `overflow:hidden`으로 **절대 안 삐져나감**. 미래 날짜는 `.is-future` 회색·클릭 불가.

### 편집기 화면 (캘린더를 대체, `← 달력` 복귀 버튼)
- 날짜: `7월 5일 토요일` (한국어 요일)
- **기분 이모지 5개** — 크고 시원한 버튼(≈1.8rem), 선택 토글
- **활동 칩**(ACTIVITY_ORDER) — 켜면 바로 아래 그 활동 `ACTIVITY_JOURNAL_FIELDS` 입력칸(거리/시간/산이름) 등장
- **일기 textarea** — placeholder "오늘 어땠나요? 코스·컨디션·느낀 점을 남겨보세요", maxLength 500
- 하단 sticky 푸터 **[취소] [저장]**
  - 저장: `onSaveEntry(date, draft)` → isEntryEmpty면 삭제, 아니면 저장 → "저장됨 ✓" 토스트 → 캘린더 복귀(`onSelectDate(null)`)
  - 취소: draft 폐기 → 캘린더 복귀

### 부모(Home) 배선 변경
- 기존 `onToggleActivity`/`onNoteChange` 제거 → **`onSaveEntry(date, entry)` + `onDeleteEntry(date)`** 로 교체. draft·필드편집은 JournalView 내부.
- 시트: `max-height: 90vh; overflow-y: auto` (내용 길어도 스크롤).
