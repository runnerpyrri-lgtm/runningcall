# v0.3.0 — 자전거 교체 · 텍스트/색상 정비 · 애견 강화 설계

- 작성일: 2026-07-05
- 상태: 승인됨 (사용자 피드백 기반)
- 범위: v0.2.0 멀티 활동의 완성도 개선. 출퇴근→자전거, 텍스트 전면 활동화, 색상 위계 재설계, 애견 모듈 강화, 선택 UI 정리.

## 배경

v0.2.0 실사용 피드백에서 5가지 문제 확인.
1. **색상 충돌** — 1등 카드(크림-앰버)와 일출 카드(크림-앰버)가 거의 같은 색이라 뭉개짐.
2. **러닝 표현 누수** — "우중런", "러닝 복장", "러닝 알림", "러닝하기 좋은" 등이 모든 카테고리에 그대로 노출.
3. **출퇴근 카테고리 부실** — 출퇴근 시간은 고정이라 "추천 시간" 개념이 약함.
4. **애견 카테고리 빈약** — 발바닥 경고 배너 하나뿐, 견주 정보 부족.
5. **선택 UI** — 데스크탑 좌측 세로 레일이 어색.

## 🔒 불변식 (유지)

- **러닝 골든마스터** — run 프로필 점수·톤·멘트는 계속 완전 불변. 리팩터 후 스냅샷 일치 필수.
- 로컬스토리지 키(`running-alarm:*`)·기존 사용자 데이터 보존. 마지막 선택 활동 기억 유지.

## 결정 사항

### 1. 출퇴근 → 자전거 교체
- `commute` 프로필 완전 제거, `bike`(자전거 🚴) 신설.
- 활동 순서: `run, walk, dog, bike`.
- **자전거 점수 프로필**
  - weights: precipitation 0.24, wind 0.22, temperature 0.18, dust 0.16, uv 0.12, humidity 0.08 (합 1.00)
  - temp: optimal 12~24°C, coldSlope 6(속도 체감), hotSlope 4
  - heat: hot1 32→60, hot2 35→40
  - **windCap 신설**: 강풍 라이딩 위험 → bike는 windSpeed ≥ 11 m/s에서 상한 38. run 등 기존은 {14, 40}으로 명시(불변 보장).
- **자전거 콘텐츠**: 동사 "라이딩". 헬멧·바람막이·장갑·아이웨어·라이트(비/야간). 맞바람 주의, 내리막 제동, 미끄럼 위험 등 자전거 전용 문구 전면 신규.

### 2. 텍스트 전면 활동화 (audit)
`ActivityProfile.terms`에 UI 재사용 문구를 담아 공용 컴포넌트가 하드코딩 대신 이걸 참조.
```
terms: {
  verbDo: "뛰"|"걷"|"산책하"|"라이딩하",
  outfitTitle: "러닝 복장"|"산책 복장"|"라이딩 복장"|"걷기 복장",
  alarmTitle: "러닝 알림"|...,
  rainyTag: "우중런"|"비 산책"|"빗속 라이딩"|"우중 걷기",
  blockedTag: "러닝 어려움"|"산책 어려움"|"라이딩 어려움"|"걷기 어려움",
  nowTag: "지금 뛰기 딱!"|...,
  notifReady: "러닝하기 좋은 시간이에요"|...
}
```
교체 대상(전부 activity 반영): 알림 본문·발동 알림, 복장 시트 제목·진입 카드, 알림 설정 제목, 알림 예약 라벨, 데이파트 점수 태그(우중런)·차단 라벨, 차트 best 라벨. 기록은 활동 무관이므로 "나의 활동 기록"으로 통일.

### 3. 애견 전용 모듈 강화 — `getDogPlan(slot)`
히어로 아래 **애견 전용 카드** 노출 (다른 활동엔 없음).
- **산책 신호등**: 가능(초록)/주의(노랑)/자제(빨강) — 점수 + 발바닥 위험 결합.
- **추천 산책 길이**: 짧게(10분내)/보통(20~30분)/길게(40분+) — 점수 기반.
- **발바닥 상태**: 기존 getPawRisk 통합 표시.
- **챙길 것**: 물·배변봉투 기본 + 더우면 그늘길, 추우면 옷 등 상황별.
- 기존 단독 발바닥 배너는 이 카드에 흡수.

### 4. 색상 위계 재설계
문제의 앰버 중복 제거, 섹션별 목적색 명확화.
- **추천 시간대 카드**: 큰 색면 대신 **좌측 액센트 바 + 흰 바탕**. 1등 골드바, 2등 실버, 3등 브론즈. 점수 숫자가 톤색을 담당. (1등의 넓은 크림-앰버 배경 제거 → 일출 카드와 구분됨)
- **일출/일몰 카드**: 일출 = 웜 피치 유지, 일몰 = 쿨 라벤더 유지 (이제 1등 카드와 안 겹침).
- **is-now(지금)**: 초록 강조 링 유지.
- 히어로(블루)·안내(의미색)·데이파트(뉴트럴+베스트 초록)·복장(화이트)와 함께 각 섹션이 시각적으로 분리되게. 대비(명도) 검증.

### 5. 선택 UI 정리
- **상단 가로 탭으로 통일** — 데스크탑도 상단 탭. 좌측 세로 레일(`.activity-rail`, `.side-left`) 제거, 좌측 컬럼은 광고만.
- ☰ 드로어 활동 선택 유지.
- 마지막 선택 활동 localStorage 기억 — 유지·검증(자전거 사용자는 재시작 시 자전거로 시작).

## 아키텍처 영향

- `lib/activity.ts` — commute 제거 + bike 추가, `terms` 필드, `windCap`, `getDogPlan`. getUmbrellaAdvice 제거.
- `lib/scoring.ts` — applyCaps에 windCap 프로필 반영(run 불변). COMMENTS commute→bike.
- `lib/insights.ts` — 모든 테이블 commute→bike로 교체(자전거 카피 작성), rainy/blocked 태그는 terms 사용하도록.
- `lib/outfit.ts` — commute→bike 복장 로직.
- `app/page.tsx` — 좌측 레일 제거, 텍스트 audit(terms 참조), 애견 모듈 렌더, 자전거 강풍 안내, 우산 안내 제거.
- `app/globals.css` — rank 카드 액센트바 재설계, sun 카드 확인, 좌측 레일 CSS 제거, 애견 모듈 스타일, 상단 탭 데스크탑 표시.
- `lib/__tests__/scoring.test.ts` — commute 테스트 → bike 테스트(강풍 시 자전거 점수 급감), 러닝 골든마스터 유지.

## 성공 기준
1. 러닝 골든마스터 계속 통과(러닝 불변).
2. 4개 활동(러닝/걷기/애견/자전거) 어디서도 다른 활동 표현이 새지 않음(grep 검증).
3. 1등 카드와 일출 카드가 시각적으로 명확히 구분.
4. 자전거: 강풍 시 점수가 러닝보다 크게 낮고, 문구가 라이딩 맞춤.
5. 애견: 신호등+길이+체크리스트 카드 표시.
6. 데스크탑/모바일 상단 탭 통일, 좌측 레일 없음, 마지막 활동 기억.
7. build/test 통과, Vercel 배포 성공.
