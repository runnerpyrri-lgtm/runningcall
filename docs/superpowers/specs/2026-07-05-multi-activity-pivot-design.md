# 나가도돼 — 멀티 활동 확장 설계 (v0.2.0~v0.5.0)

- 작성일: 2026-07-05
- 상태: 승인됨 (사용자 + 외부 AI + 코드분석 3자 합의)
- 범위: 러닝 전용 → "야외활동 컨디션 앱" 리브랜딩 + 활동 카테고리 4종

## 배경 / 결정

러닝콜은 러너에겐 완성형이지만 "러닝"이라는 이름이 입구에서 시장을 좁힌다.
걷기·산책·반려견·출퇴근 도보까지 포용하면 수요가 훨씬 크다(NCHS/CDC: 미국 성인 58.7%가 최근 7일 내 여가 걷기 / APPA 2025: 반려견 보유 7,100만 가구).
**러닝을 갈아엎지 않고**, "나가도돼"라는 큰 앱 안에 러닝을 프리미엄 카테고리로 넣는다.

## 🔒 최우선 불변식 (Non-negotiable)

1. **러닝 카테고리는 현재 완성형을 100% 보존한다.**
   - `run` 프로필 = 현재 `scoring.ts`의 가중치·온도곡선·하드캡·멘트를 값 그대로 이관.
   - `calculateRunningSlot(input)`은 삭제하지 않고 `calculateSlot(input, ACTIVITIES.run)` 별칭으로 유지.
   - **검증**: 리팩터 전 러닝 점수(대표 지점·48시간)를 스냅샷 → 리팩터 후 완전 일치 확인. 불일치 시 배포 금지.
   - 러닝 선택 시 UI는 현재와 시각적으로 동일.
2. **UI/UX 일관성**: 히어로·추천 시간대·복장·한마디·차트·시트 뼈대는 그대로. 카테고리에 따라 **내용(문구·점수·경고)만** 바뀐다. 새 카테고리도 러닝과 같은 컴포넌트·같은 톤·같은 색 체계를 쓴다.

## 브랜드

- 앱 이름: **나가도돼** (1순위. 도메인 가용성은 도메인 단계에서 조회)
- 메인 질문: **"지금 나가도 돼?"**
- 서브 태그라인: **"걷기·산책·러닝, 지금 나가기 좋은 시간"**
- 배포 주소는 `runningcall.vercel.app` 유지(불변). 새 도메인은 나중 단계.

## 카테고리 4종

| 키 | 라벨 | 이모지 | 성격 |
|---|---|---|---|
| `run` | 러닝 | 🏃 | 현재 완성형 그대로 (불변) |
| `walk` | 걷기 | 🚶 | 일반 유저용. 더위·습도 관대, 미세·자외선 강조 |
| `dog` | 애견산책 | 🐕 | 킬러. 지면열/발바닥 화상/한낮 위험 경고 |
| `commute` | 출퇴근 | ☂️ | 점수보다 "비 맞을까/우산 필요?" 정보 우선 |

### 활동별 점수 방향 (구체 수치)

**러닝 (run) — 현재값, 변경 없음**
- 최적 체감 8~16°C, 더위/습도/강수 민감, 추천 기준 빡세게(각 55↑/평균 62↑)

**걷기 (walk)**
- 최적 체감 10~22°C (더위 관대), 더위 상한 발동 33°C→60 / 36°C→40
- 가중치: 미세 0.28→0.30, 자외선 0.12→0.16, 습도 0.11→0.07, 바람 0.11→0.08
- 톤: "뛰기 어려움" 대신 "짧은 산책 가능"

**애견산책 (dog)** — 걷기 기반 + 경고 레이어
- 낮 11~16시 & 기온 ≥26°C & 자외선 높음 → **강한 경고 배너** ("발바닥 뜨거울 수 있어요", "그늘길 짧게가 좋아요")
- 미세먼지 가중 더 ↑(강아지 호흡기)
- 결과 표현: 점수보다 **산책 가능 / 주의 / 피하기** 3단계 우선

**출퇴근 (commute)** — 걷기 기반 + 비 회피 중심
- 강수확률·강수량 가중 최상, 미세·자외선은 보조
- 결과 표현: 점수보다 **"우산 필요" / "20분 안에는 괜찮음" / "퇴근길 비 가능"**

## 아키텍처

### 신규: `lib/activity.ts`
```
export type ActivityKey = "walk" | "run" | "dog" | "commute";
export type ActivityProfile = {
  key; label; emoji; verb;
  weights;              // 지표별 가중치
  tempOptimal: [n, n];  // 최적 체감온도
  heatCap;              // 더위 상한 발동값
  phrases;              // 헤드라인/서브라인/멘트/한마디 세트
  resultMode;           // "score" | "advisory"(애견·출퇴근)
};
export const ACTIVITIES: Record<ActivityKey, ActivityProfile>;
```

### `lib/scoring.ts` (line 35 WEIGHTS / 104 온도 / 166 하드캡)
- 하드코딩 상수를 `profile`에서 읽도록 파라미터화.
- `calculateSlot(input, profile)` 신설, `calculateRunningSlot` = run 별칭 유지.
- `COMMENTS`(205)를 프로필로 이동.

### `lib/weather.ts` (line 127) — 원본/점수 분리
- 지금은 fetch 직후 `calculateRunningSlot`으로 고정 계산.
- **원본 HourlyInput[]만 반환**하도록 분리 → 점수화는 클라이언트에서.
- 효과: 카테고리 전환 시 **재요청 없이 즉시 재계산**.

### `lib/insights.ts` (line 197~) — 콘텐츠 활동화 (작업량 최대)
- `heroHeadline`·`heroSubline`·`composeOneLiner`·`composeBriefing`·`getMetricDetail`(meaning/tip)·`getConditionChips`·`getRankedWindows`에 `profile` 인자 추가.
- 러닝 표현("뛰기/페이스/조깅") → 활동별 카피 세트로 분리.
- 예: 걷기 "가볍게 걷기 좋아요" / 러닝 "지금 뛰기 좋아요" / 애견 "산책은 저녁이 좋아요" / 출퇴근 "지금 나가면 비는 피할 수 있어요".

### `lib/outfit.ts` — 복장 활동화 (`profile` 인자)

### `app/page.tsx` (Home, line 948~ / 히어로 1768~)
- `activity` state + localStorage `running-alarm:activity`.
- slots = `useMemo(rawHourly, ACTIVITIES[activity])`.
- 신규 `<ActivityRail>` 컴포넌트:
  - **데스크탑**: 좌측 세로 카테고리 레일 (현재 좌측 광고 슬롯 `AdSlot side="left"` 자리를 대체). 🚶걷기 / 🏃러닝 / 🐕애견산책 / ☂️출퇴근.
  - **모바일**: 히어로 위 가로 탭.
  - ☰ 드로어 메뉴에도 동일 카테고리 노출.
- 상단 브랜드 워드마크 "러닝콜" → "나가도돼", 서브 태그라인.
- 애견·출퇴근은 `resultMode: "advisory"`면 점수 링 대신 상태(가능/주의/피하기, 우산 필요 등) 우선 표기 — 단 컴포넌트·색·레이아웃은 러닝과 통일.

## 리스크 / 엣지케이스

- 러닝 회귀(값 변동): 스냅샷 검증으로 차단.
- advisory 모드가 기존 점수 UI와 충돌 → 같은 컴포넌트 재사용, 표기만 스위칭.
- 카테고리 전환 시 열린 시트/알람 상태 꼬임 → 전환 시 시트 닫고 재계산.
- localStorage에 기존 러닝 사용자 데이터(위치/알람/기록) 유지, 활동 키만 추가.
- 문구 이관 누락으로 러닝 표현이 걷기에 새어나옴 → 카피는 100% 프로필에서만 조회.

## 성공 기준

1. 러닝 카테고리: 리팩터 전후 점수 스냅샷 완전 일치 + 화면 동일.
2. 걷기: 같은 날씨에 러닝보다 더위/습도 관대한 점수가 실제로 나옴.
3. 카테고리 전환이 재요청 없이 즉시(<100ms) 반영.
4. 4개 카테고리 모두 러닝과 동일한 UI 톤·색·레이아웃 일관성.
5. `pnpm build`/`typecheck` 통과, Vercel 자동배포 성공.
