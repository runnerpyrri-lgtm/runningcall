# 나가도돼 v0.2.0 (멀티 활동: 러닝 + 걷기) 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 러닝 전용 앱을 "나가도돼"로 리브랜딩하고, 좌측 카테고리 레일로 러닝·걷기 2종을 전환하며, 걷기는 걷기 전용 점수·문구로 재계산되게 한다. (애견·출퇴근 탭은 "준비 중" 자리만 노출)

**Architecture:** 점수 계산을 활동 프로필(`lib/activity.ts`) 기반으로 파라미터화한다. API는 원본 날씨(raw)만 반환하고, 클라이언트가 선택된 활동 프로필로 점수를 즉시 재계산한다. `run` 프로필은 현재 상수를 값 그대로 이관해 러닝 결과를 100% 보존한다(골든마스터 테스트로 잠금).

**Tech Stack:** Next.js 16, React 19, TypeScript, Vitest(신규, 스코어링 검증용), Open-Meteo API.

## Global Constraints

- **러닝 불변**: `run` 프로필 = 현재 `scoring.ts` 상수 값 그대로. `calculateRunningSlot`은 삭제 금지, `calculateSlot(input, ACTIVITIES.run)` 별칭으로 유지. 골든마스터 스냅샷 완전 일치 아니면 배포 금지.

```ts
export function scoreTemperature(apparent: number, temp = ACTIVITIES.run.temp) {
  if (apparent >= temp.optimalLo && apparent <= temp.optimalHi) return 100;
  if (apparent < temp.optimalLo) return clampScore(100 - (temp.optimalLo - apparent) * temp.coldSlope);
  return clampScore(100 - (apparent - temp.optimalHi) * temp.hotSlope);
}
```
3. `applyCaps`에 heat 프로필 반영 (더위 2단계만 프로필화, 나머지 캡은 공통 유지):
```ts
function applyCaps(total: number, input: HourlyInput, heat = ACTIVITIES.run.heat) {
  let cap = 100;
  if (input.precipitation >= 4) cap = Math.min(cap, 25);
  else if (input.precipitation >= 1) cap = Math.min(cap, 45);
  else if (input.precipitationProbability >= 80) cap = Math.min(cap, 60);
  if (input.apparentTemperature >= heat.hot2) cap = Math.min(cap, heat.hot2Cap);
  else if (input.apparentTemperature >= heat.hot1) cap = Math.min(cap, heat.hot1Cap);
  if (input.apparentTemperature <= -10) cap = Math.min(cap, 35);
  else if (input.apparentTemperature <= -5) cap = Math.min(cap, 55);
  if (input.pm25 > 75 || input.pm10 > 150) cap = Math.min(cap, 30);
  if (input.windSpeed >= 14) cap = Math.min(cap, 40);
  if (input.uvIndex >= 11) cap = Math.min(cap, 50);
  return Math.min(total, cap);
}
```

5. `calculateSlot` 신설 + `calculateRunningSlot` 별칭:
```ts
export function calculateSlot(input: HourlyInput, profile: ActivityProfile): RunningSlot {
  const dustScore = scoreDust(input.pm25, input.pm10);
  const uvScore = scoreUv(input.uvIndex);
  const temperatureScore = scoreTemperature(input.apparentTemperature, profile.temp);
  const humidityScore = scoreHumidity(input.humidity);
  const precipitationScore = scorePrecipitation(input.precipitation, input.precipitationProbability);
  const windScore = scoreWind(input.windSpeed);
  const w = profile.weights;
  const weighted =
    dustScore * w.dust + temperatureScore * w.temperature + precipitationScore * w.precipitation +
    uvScore * w.uv + humidityScore * w.humidity + windScore * w.wind;
  const totalScore = clampScore(applyCaps(weighted, input, profile.heat));
  const hour = Number(input.time.slice(11, 13));
  const day = Number(input.time.slice(8, 10));
  const seed = day * 31 + hour;
  return {
    ...input, hour, totalScore, dustScore, uvScore, temperatureScore, humidityScore,
    precipitationScore, windScore, comment: scoreComment(totalScore, profile.key, seed), tone: scoreTone(totalScore)
  };
}
export function calculateRunningSlot(input: HourlyInput): RunningSlot {
  return calculateSlot(input, ACTIVITIES.run);
}
```

- [ ] **Step 1: weather.ts 리팩터**

`fetchRunningForecast` 내부의 fetch+파싱까지는 유지하되, `allSlots = times.map(... calculateRunningSlot)` 부분을 `allInputs = times.map(... input)` (점수화 제거)로 바꾸고, 날짜 그룹핑을 raw input 기준으로 수행하여 `RawForecast`를 반환하는 `fetchRawForecast`로 분리한다. 그리고:
```ts
import { calculateSlot, type HourlyInput, type RunningSlot } from "@/lib/scoring";
import { ACTIVITIES, type ActivityProfile } from "@/lib/activity";

export type RawForecast = {
  location: LocationPoint; timezone: string; generatedAt: string;
  today: HourlyInput[]; yesterday: HourlyInput[]; tomorrow: HourlyInput[];
  sunrise: string | null; sunset: string | null;
};

export function scoreForecast(raw: RawForecast, profile: ActivityProfile): RunningForecast {
  return {
    location: raw.location, timezone: raw.timezone, generatedAt: raw.generatedAt,
    slots: raw.today.map((i) => calculateSlot(i, profile)),
    yesterday: raw.yesterday.map((i) => calculateSlot(i, profile)),
    tomorrow: raw.tomorrow.map((i) => calculateSlot(i, profile)),
    sunrise: raw.sunrise, sunset: raw.sunset
  };
}

export async function fetchRunningForecast(location: LocationPoint): Promise<RunningForecast> {
  return scoreForecast(await fetchRawForecast(location), ACTIVITIES.run);
}
```
`fetchRawForecast`는 기존 fetch+그룹핑을 그대로 쓰되 `HourlyInput[]`을 그룹핑해 `today/yesterday/tomorrow`로 반환한다(점수화 없음).
