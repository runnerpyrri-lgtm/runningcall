# 야외봄 전체 점검 및 AI 교차 검토

## 점검 기준

- 기준 브랜치: `agent/mobile-primary-action`, Draft PR #11.
- 비교 기준: `main`과 PR 브랜치를 390×844, 360×800, 1440×900에서 직접 확인.
- 모바일 실측: CTA는 390×844에서 y=658→543, 360×800에서 y=649→533으로 올라왔다.
- 터치 실측: CTA 54px, 활동·위치·날짜 44px, 지표 46px. 준비 버튼은 32px로 남아 있다.
- 브라우저 상태: 가로 넘침과 Next.js 오류 오버레이, 콘솔 오류가 없었다.
- 직접 재현: 추천 다이얼로그를 열어도 포커스가 배경 CTA에 남고, `opacity: 0`인 미공개 결과 버튼이 `disabled=false`, `tabIndex=0`으로 노출됐다.

## 독립 담당자 1차 분석

### 모바일 UX·접근성 담당

- CTA 이동과 44px 보정은 첫 화면 도달성과 한 손 조작을 개선하므로 채택 의견.
- `TimeReel`의 초기 포커스, 포커스 트랩·복귀, Escape, 미공개 결과 비활성화, 명시적 건너뛰기·닫기를 병합 전 필수로 제안.
- 준비 버튼과 위치 모달 핵심 컨트롤의 44px 보정, 로딩·오류 라이브 영역을 추가 제안.
- 고정 CTA는 화면을 가리고 기존 fixed 요소와 충돌할 수 있어 제외 의견.

### 성능·PWA·안정성 담당

- CTA 이동 자체의 상태 회귀는 낮다고 판단.
- 위치 변경 시 이전 예보 요청이 최신 위치 결과를 덮을 수 있는 경합을 P0으로 발견.
- PWA 등록·캐시 수명주기, 역지오코딩 시간 예산, 오프라인 마지막 예보를 후속 후보로 제안.
- 추천 계산이 CTA 노출과 실제 릴 생성에서 중복되는 점을 낮은 비용의 개선으로 제안.

### 코드·테스트·회귀 담당

- `TimeReel` 접근성과 `getRankedWindows` 경계 테스트를 병합 전 보강할 가치가 크다고 판단.
- 추천 계산과 CTA 마크업의 중복을 좁게 정리하되 거대 파일 분리는 보류 의견.
- 날짜 선택의 불완전한 탭 의미론을 발견했지만, 이번 PR에 넣을지는 낮은 우선순위로 분류.

## Claude Code 교차 검토

Claude Code는 PR #11 실제 diff와 관련 코드를 독립 확인하고 다음과 같이 분류했다.

- 채택: CTA 이동, 다이얼로그 접근성, 로딩·오류 라이브 영역, 추천 계산·CTA 정의 일원화, 핵심 44px 보정, `getRankedWindows` 테스트.
- 보완: CTA 노출 판단은 실제 `reelRanks` 결과와 같은 파생값을 사용해야 한다. 현재 `hasRecommendedWindow`와 `reelRanks.length`의 조건이 달라 CTA가 보이지만 릴이 열리지 않는 경계가 가능하다.
- 반박·보류: 날짜 탭 의미론은 별도 접근성 범위가 적절하다. 위치 예보 경합은 실재하지만 UI PR에 섞지 말고 별도 안정성 PR로 분리한다. PWA와 역지오코딩 변경도 이번 CTA 범위에서는 보류한다.

## 담당자 교차 검토

- UX 담당은 Claude의 5개 범위에 동의하되, 다이얼로그 접근성은 포커스 이동만이 아니라 명시적 `결과 바로 보기`와 `닫기`까지 포함해야 한다고 보완했다.
- 안정성 담당은 forecast 경합을 별도 PR로 분리하는 데 동의했다. 다만 미공개 결과 버튼 비활성화는 포커스 트랩과 별개인 필수 조치라고 강조했다.
- 코드 담당은 JS로 모바일 여부를 판별해 CTA 하나만 렌더링하는 방식에 반대했다. CSS 반응형 배치는 유지하고 계산·문구·동작만 공통화해야 hydration과 회전 회귀를 피할 수 있다고 보완했다.
- 세 담당자 모두 44px 확대는 준비·닫기·핵심 실행 버튼으로 제한하고, 360×800에서 CTA 위치를 다시 측정해야 한다고 합의했다.

## 최종 구현 범위

1. 기존 CTA 이동과 주요 44px 변경을 유지한다.
2. `TimeReel`에 초기 포커스, 포커스 트랩·복귀, Escape, 명시적 건너뛰기·닫기, 라이브 상태, 미공개 결과 비활성화를 추가한다.
3. 추천 결과를 한 번 파생해 CTA 노출·릴 데이터가 같은 기준을 공유하고, CTA 마크업을 공통화한다. CSS 반응형 배치는 유지한다.
4. 로딩과 치명적 오류에 라이브 의미론을 추가하고, 준비·위치 모달의 핵심 터치 타깃만 44px로 보정한다.
5. `getRankedWindows`의 CTA 관련 경계 단위 테스트를 추가한다.

forecast 경합, PWA, 빈 상태 행동, 날짜 탭, 대규모 분리는 이번 PR에서 구현하지 않는다.

## 실제 반영 결과

- 모바일 CTA는 히어로 직후에 유지하고 CTA 표시와 릴 데이터가 동일한 `recommendation` 파생값을 사용하도록 일원화했다.
- `TimeReel`에 초기 포커스, Tab 포커스 순환, Escape 닫기, 닫은 뒤 CTA 포커스 복귀, 명시적 결과 바로 보기·닫기, 공개 전 결과 비활성화, 진행 상태 라이브 알림을 추가했다.
- 준비, 시트·위치 모달 닫기, 검색, 저장 위치 탭, 빠른 검색의 핵심 터치 영역을 44px로 보정했다.
- 로딩·치명적 오류·위치 확인·검색·토스트 상태에 보조기기 알림 의미론을 추가했다.
- `getRankedWindows`에 오늘의 지난 시간, 내일, 강수·점수 임계값, 등산 새벽 경계 테스트를 추가했다.

## 구현 후 검증

- `pnpm typecheck`: 통과.
- `pnpm test`: 7개 파일, 53개 테스트 통과.
- `pnpm build`: Next.js 16.2.10 프로덕션 빌드 통과.
- 360×800: CTA y=545, 높이 54px, 준비 버튼 44px, 가로 넘침·오류 오버레이·콘솔 오류 없음.
- 390×844: CTA y=555, 높이 54px, 준비 버튼 44px, 가로 넘침 없음.
- 1440×900: 데스크톱 CTA는 우측 열 y=168, 높이 54px로 기존 2단 배치를 유지.
- 위치 모달: 닫기·검색·저장 위치 탭·빠른 검색이 모두 44px, 390×844에서 모달 전체가 스크롤 없이 표시.
- 추천 다이얼로그: 열린 뒤 `닫기`로 포커스 이동, Tab으로 `결과 바로 보기` 순환, Escape로 닫고 CTA로 포커스 복귀.
- 미공개 결과: `disabled=true`, `aria-hidden=true`, opacity 0. 결과 바로 보기 후 `disabled=false`, `aria-hidden=false`, opacity 1.
- PWA 산출물: 프로덕션 서버에서 루트, manifest, service worker, 192/512/maskable 아이콘이 모두 HTTP 200.
